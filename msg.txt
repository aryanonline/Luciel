WHY (T2 split: Commit 8 -> 8a + 8b)

Commit 8 was originally scoped as a single commit covering both
artifact production and prod-side AWS execution. After tonight's
recon surfaced four sequential naming/state surprises (cluster name
luciel-cluster, service names luciel-backend-service /
luciel-worker-service, Postgres role canonical form luciel_worker
with underscore, VPC endpoint SG ingress requirement), the
combined-commit scope grew to ~75 min of new AWS state changes at
the end of a 22-hour day.

Splitting into 8a (artifacts, this commit) + 8b (prod execution,
tomorrow morning fresh) preserves tonight's recon momentum into
reviewable code/docs while deferring all irreversible AWS calls
(SG creation, RDS role password change, task-def revision, service
update) to rested-judgment morning execution.

Brokerage tech-due-diligence posture: the mint script + runbook
together are the auditable artifact for "how does VantageMind
provision worker DB credentials in production?" -- both are
reviewable in source control before any AWS state changes happen.
SOC 2 alignment: written policy enforced by tooling, no exceptions.
